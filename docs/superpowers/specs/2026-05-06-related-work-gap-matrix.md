# Related-Work Gap Matrix

## Positioning Rule

For each paper below, keep the comparison tied to the thesis main line: KG-grounded, contract-bounded, executable geospatial fusion runtime with auditable evidence.

## Geo-Agent

- Does well: natural-language GIS interaction and spatial intent decomposition.
- FusionAgent already exceeds: executable artifact output, audit contract, bounded runtime stages.
- FusionAgent still lacks: richer interactive spatial UX and broader user-facing GIS affordances.
- Borrow without drift: spatial intent decomposition and graph-aware retrieval.

## CyVerACT

- Does well: schema-aware deterministic verification and structured error reasoning.
- FusionAgent already exceeds: end-to-end execution and artifact evidence instead of only query verification.
- FusionAgent still lacks: uniform tool-contract and failure-taxonomy enforcement.
- Borrow without drift: deterministic verification loops and machine-readable error metadata.

## OntoLLM

- Does well: ontology-grounded reasoning constraints and digression prevention.
- FusionAgent already exceeds: task and algorithm execution grounded in KG-facing runtime entities.
- FusionAgent still lacks: stronger unsupported-intent rejection and ontology-backed boundary guards.
- Borrow without drift: ontology-grounded rejection and bounded reasoning discipline.

## PathMind

- Does well: path prioritization and low-noise retrieval.
- FusionAgent already exceeds: explicit execution, healing, and evidence writeback.
- FusionAgent still lacks: retrieval prioritization transparency and token-efficiency discipline.
- Borrow without drift: path prioritization and low-noise candidate ranking.

## UniAI-GraphRAG

- Does well: dual-channel retrieval, ontology-guided extraction, and report-oriented synthesis.
- FusionAgent already exceeds: executable geospatial artifact production instead of text-answer endpoints.
- FusionAgent still lacks: clearer retrieval summaries and community-level evidence reporting.
- Borrow without drift: dual-channel retrieval summaries and structured extraction patterns.

## 地下车站洪水脆弱性级联效应分析

- Does well: facility dependency and flood cascade reasoning.
- FusionAgent already exceeds: bounded runtime orchestration and evidence contract across task slices.
- FusionAgent still lacks: richer scenario dependency reasoning depth.
- Borrow without drift: bounded facility dependency and cascade reasoning for scenario profiles.
