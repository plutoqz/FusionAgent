# Redundancy And Drift Ledger

## Drift Item

- Item: artifact reuse branches
- Risk: artifact reuse branches keep accumulating implementation paths that are useful but easy to overclaim as the main runtime differentiator.
- Action: keep but demote. Preserve the current reuse paths, but keep the claim priority on contract-bounded planning, execution, and evidence.

## Drift Item

- Item: documentation duplication
- Risk: README, operations docs, plans, and freeze docs restate the same boundaries with slightly different wording and drift over time.
- Action: keep and strengthen. Use boundary tests and the capability inventory as the canonical wording anchors.

## Drift Item

- Item: Benin script sprawl
- Risk: research utilities and cleanup scripts start replacing runtime interfaces in the repo narrative.
- Action: keep but demote. Mark one-off Benin preparation, cleanup, and benchmarking scripts as research utilities unless the capability is frozen through shared runtime evidence.

## Drift Item

- Item: trajectory-to-road wording
- Risk: a metadata seam gets misread as an executable runtime capability.
- Action: freeze. Keep reservation-only wording everywhere until executable runtime evidence, tests, and operations wording all exist.

## Drift Item

- Item: durable learning wording
- Risk: bounded evidence aggregation gets overstated as autonomous self-evolution.
- Action: keep and strengthen. Describe durable learning as bounded policy hints until a later phase adds explicit policy consumption evidence.

## Drift Item

- Item: front-end evidence surface
- Risk: UI polish drifts into the main system claim before operator evidence surfaces are complete.
- Action: keep but demote. The web workbench can remain as operator support, but it must not replace no-UI evidence and inspection surfaces in the core claim.
