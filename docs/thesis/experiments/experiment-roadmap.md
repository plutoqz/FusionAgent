# Thesis Experiment Roadmap

This roadmap translates the thesis priorities into concrete experiment work. It separates executable work that can be run now from claims that must wait for frozen metrics.

## Priority 1: Evidence Refresh

| Task | Command or source | Output | Promotion rule |
| --- | --- | --- | --- |

## Priority 2: Automatic Fusion Experiments

| Experiment | Task family | Purpose | Current evidence use |
| --- | --- | --- | --- |
| Building full-system run | `building` | Show KG-grounded planning, validation, execution, artifact emission, and evidence writeback. | Main `C1/C2` support. |
| Road runtime smoke | `road` | Show shared runtime behavior beyond building. | Supporting bounded extensibility. |
| Water runtime smoke | `water` | Show task-driven runtime and evidence contract on water slices. | Supporting bounded extensibility. |
| Bounded POI runtime smoke | `bounded_poi` | Show the bounded POI path without broadening task claims. | Supporting only. |

Required outputs for each promoted run:

- `run.json`
- `plan.json`
- `validation.json`
- `audit.jsonl`
- artifact bundle
- inspection or report summary when available

## Priority 3: Ablation Experiments

| Ablation | Comparison | Primary metrics | Claim |
| --- | --- | --- | --- |
| A0 vs full runtime | Coarse task-to-handler runtime vs KG-decomposed planning | `unknown_algorithm_rate`, `planning_valid_rate`, `execution_success_rate` | `C1` |
| Weak grounding vs full runtime | Reduced KG visibility or monolithic algorithm selection vs full runtime | `validator_rejection_rate`, `kg_fallback_rate`, `final_executable_success_rate` | `C1/C2` |
| No healing vs healing | Disable repair/replan path vs bounded recovery path | `recovery_success_rate`, `policy_sourced_repair_count`, `execution_success_rate` | `C3` |
| Result-only vs evidence contract | Result artifact only vs full run-level evidence bundle | `evidence_completeness_rate`, `decision_trace_completeness`, inspection availability | `C2` |

Do not promote any ablation result until the output directory is frozen with `scripts/freeze_experiment_evidence.py`.

Current frozen evidence: `exp-ablation-a0-a2-controlled-comparison` freezes a full controlled A0/A1/A2a/A2b/A2c comparison. A2c rows are copied from inspection-derived full-runtime evidence; A0, A1, A2a, and A2b rows are deterministic counterfactual rows generated by `scripts/materialize_controlled_ablation_rows.py`. Use it for controlled ablation discussion and keep the counterfactual label visible.

## Priority 4: Fusion Quality Experiments

| Area | Metrics | Thesis role |
| --- | --- | --- |
| Geometry validity | invalid geometry rate, empty geometry rate, readable output status | Distinguishes execution success from GIS output quality. |
| Duplication and conflation behavior | duplicate geometry rate, retained feature count, source contribution balance | Supports quality discussion without overstating agent intelligence. |
| Task-family metrics | building/road/water/POI-specific checks where available | Used in benchmark or appendix after Freeze B alignment. |
| Controlled semi-real supplement | configured threshold pass/fail plus artifact lineage | Supports a bounded robustness discussion, not a real Benin quality claim. |

Current frozen supplement: `exp-quality-freeze-b-controlled-supplement` materializes four deterministic semi-real artifacts, runs the quality benchmark, and freezes the outputs in `docs/superpowers/specs/2026-06-10-freeze-c-quality-controlled-supplement-manifest.json`.

Original Freeze B readiness: `runs/experiments/exp-quality-freeze-b/readiness.json` reports that five non-smoke Freeze B cases are blocked by missing `precomputed_artifact_path` values. Real-world quality metrics should stay out of final Chapter 5 claims until those artifacts are materialized and frozen.
