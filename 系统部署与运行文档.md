# GeoFusion 系统部署与运行文档

## 一、系统环境要求

### 1.1 操作系统要求

| 操作系统 | 推荐版本 | 最低版本 |
|---------|---------|---------|
| Linux | Ubuntu 20.04+ / Debian 11+ | Ubuntu 18.04 |
| Windows | Windows 10/11 (WSL2) | Windows 10 |
| macOS | macOS 12+ | macOS 11 |

**注意**：推荐使用 Linux 发行版以获得最佳性能和稳定性。Windows 用户建议通过 WSL2 运行。

### 1.2 Python 环境

| 组件 | 推荐版本 | 最低版本 |
|-----|---------|---------|
| Python | 3.9 - 3.11 | 3.9 |

建议使用虚拟环境管理依赖：
```bash
python -m venv venv
# Linux/macOS
source venv/bin/activate
# Windows
venv\Scripts\activate
```

### 1.3 依赖软件

| 软件 | 用途 | 版本要求 | 备注 |
|-----|-----|---------|-----|
| Redis | Celery 消息队列 | 6.0+ | 本地调试可使用内置 Eager 模式 |
| Neo4j | 知识图谱后端 | 5.x | 可选，v2 功能必需 |
| Node.js | 前端依赖(如有) | 16+ | 仅当前端项目存在时需要 |

### 1.4 硬件配置建议

| 配置项 | 最低配置 | 推荐配置 |
|-------|---------|---------|
| CPU | 2 核 | 4 核及以上 |
| 内存 | 4 GB | 8 GB 及以上 |
| 磁盘 | 20 GB | 50 GB 及以上 |
| 网络 | 10 Mbps | 100 Mbps |

### 1.5 Python 依赖包

所有依赖已包含在 `requirements.txt` 中，安装命令：
```bash
pip install -r requirements.txt
```

核心依赖列表：
- **Web 框架**: fastapi>=0.111.0, uvicorn>=0.30.0
- **数据处理**: geopandas>=0.14.0, shapely>=2.0.0, pandas>=2.0.0
- **任务队列**: celery>=5.4.0, redis>=5.0.0
- **图数据库**: neo4j>=5.20.0
- **测试**: pytest>=8.0.0

## 二、密钥配置指南

### 2.1 环境变量配置方式

系统通过**环境变量**加载配置。建议创建 `.env` 文件存放配置（需自行创建，项目未提供模板），文件放在项目根目录。

### 2.2 密钥配置项

#### 2.2.1 知识图谱配置 (Neo4j)

| 环境变量 | 必填 | 说明 | 示例值 |
|---------|-----|-----|-------|
| `GEOFUSION_KG_BACKEND` | 否 | 知识图谱后端类型，可选 `neo4j` 或 `memory`，默认 `neo4j` | `neo4j` |
| `GEOFUSION_NEO4J_URI` | 仅后端为 neo4j 时 | Neo4j 连接地址 | `bolt://localhost:7687` |
| `GEOFUSION_NEO4J_USER` | 仅后端为 neo4j 时 | Neo4j 用户名 | `neo4j` |
| `GEOFUSION_NEO4J_PASSWORD` | 仅后端为 neo4j 时 | Neo4j 密码 | `your_password` |
| `GEOFUSION_NEO4J_DATABASE` | 否 | Neo4j 数据库名 | `neo4j` |

**获取途径**：
1. 安装 Neo4j Desktop 或使用 Neo4j Aura 云服务
2. 创建数据库并设置密码
3. 获取连接 URI

**注意**：如未配置 Neo4j 或配置不完整，系统将自动回退到内存模式（仅适合本地调试）。

#### 2.2.2 LLM (大语言模型) 配置

| 环境变量 | 必填 | 说明 | 示例值 |
|---------|-----|-----|-------|
| `GEOFUSION_LLM_PROVIDER` | 否 | LLM 提供商，可选 `auto`、`openai`、`mock`，默认 `auto` | `openai` |
| `OPENAI_API_KEY` | 使用 OpenAI 时 | OpenAI API 密钥 | `sk-...` |
| `GEOFUSION_LLM_API_KEY` | 否 | LLM API 密钥（优先级高于 OPENAI_API_KEY） | `sk-...` |
| `GEOFUSION_LLM_MODEL` | 否 | 模型名称，默认 `gpt-4.1-mini` | `gpt-4.1` |
| `GEOFUSION_LLM_BASE_URL` | 否 | API 端点 URL，默认 `https://api.openai.com/v1` | `https://api.openai.com/v1` |
| `GEOFUSION_LLM_TIMEOUT_SEC` | 否 | 请求超时时间（秒），默认 60 | `60` |

**获取途径**：
1. 访问 [OpenAI Platform](https://platform.openai.com/) 注册账号
2. 在 API Keys 页面创建密钥
3. 确保账户有足够的 API 调用配额

**注意**：
- 使用 `mock` 提供商可进行本地测试（不调用真实 LLM）
- v1 API (融合功能) 不需要 LLM 配置

#### 2.2.3 Celery 任务队列配置

| 环境变量 | 必填 | 说明 | 默认值 |
|---------|-----|-----|-------|
| `GEOFUSION_CELERY_BROKER` | 否 | Redis 连接地址 | `redis://localhost:6379/0` |
| `GEOFUSION_CELERY_BACKEND` | 否 | Celery 结果后端 | 同 broker |
| `GEOFUSION_CELERY_EAGER` | 否 | 是否使用 Eager 模式（本地调试） | `1` |
| `GEOFUSION_TIMEZONE` | 否 | 时区设置 | `Asia/Shanghai` |

**说明**：
- `GEOFUSION_CELERY_EAGER=1` 时，任务同步执行，无需启动 Redis（推荐本地开发使用）
- 生产环境需设置为 `0` 并配置 Redis

### 2.3 配置示例 (.env 文件)

```bash
# 知识图谱 (可选，如不配置将使用内存模式)
GEOFUSION_KG_BACKEND=neo4j
GEOFUSION_NEO4J_URI=bolt://localhost:7687
GEOFUSION_NEO4J_USER=neo4j
GEOFUSION_NEO4J_PASSWORD=your_secure_password
GEOFUSION_NEO4J_DATABASE=neo4j

# LLM (v2 功能必需)
GEOFUSION_LLM_PROVIDER=openai
OPENAI_API_KEY=sk-your-openai-api-key
GEOFUSION_LLM_MODEL=gpt-4.1-mini

# Celery (本地调试使用 Eager 模式)
GEOFUSION_CELERY_EAGER=1
GEOFUSION_TIMEZONE=Asia/Shanghai
```

**重要安全提醒**：
- 切勿将包含真实密钥的 `.env` 文件提交到版本控制系统
- 已在 `.gitignore` 中排除 `.env` 文件
- 生产环境请使用环境变量注入或密钥管理服务

## 三、系统启动步骤

### 3.1 环境准备

#### 步骤 1：克隆或获取项目代码
```bash
cd /path/to/project
```

#### 步骤 2：创建并激活虚拟环境
```bash
python -m venv venv

# Linux/macOS
source venv/bin/activate

# Windows (PowerShell)
.\venv\Scripts\Activate

# Windows (CMD)
venv\Scripts\activate.bat
```

#### 步骤 3：安装依赖
```bash
pip install -r requirements.txt
```

### 3.2 配置密钥

根据上一节指南，创建 `.env` 文件并配置必要的环境变量。

### 3.3 启动服务

#### 方式一：本地开发模式（推荐）

只需启动 FastAPI 应用：
```bash
uvicorn main:app --reload --host 0.0.0.0 --port 8000
```

- `--reload`: 代码变更时自动重载
- `--host 0.0.0.0`: 允许外部访问
- `--port 8000`: 服务端口

#### 方式二：生产模式

```bash
# 使用 gunicorn 作为生产服务器
pip install gunicorn

gunicorn main:app -w 4 -k uvicorn.workers.UvicornWorker --bind 0.0.0.0:8000
```

#### 方式三：后台运行

```bash
# Linux/macOS
nohup uvicorn main:app --host 0.0.0.0 --port 8000 > app.log 2>&1 &

# Windows (PowerShell)
Start-Process -FilePath "uvicorn" -ArgumentList "main:app --host 0.0.0.0 --port 8000" -WindowStyle Hidden
```

### 3.4 启动 Redis (生产环境)

如果 `GEOFUSION_CELERY_EAGER=0`，需要启动 Redis：
```bash
# 使用 Docker
docker run -d -p 6379:6379 --name redis redis:7-alpine

# 或直接安装
# Ubuntu/Debian
sudo apt install redis-server
sudo systemctl start redis

# Windows (WSL)
sudo apt install redis-server
```

### 3.5 启动 Neo4j (可选)

如需使用知识图谱功能：
```bash
# 使用 Docker
docker run -d -p 7474:7474 -p 7687:7687 \
  -e NEO4J_AUTH=neo4j/your_password \
  --name neo4j neo4j:5

# 或使用 Neo4j Desktop
# 下载地址: https://neo4j.com/download/
```

## 五、注意事项

### 5.1 端口占用处理

- **默认端口**: 8000
- **查看端口占用**:
  ```bash
  # Linux
  lsof -i :8000
  
  # Windows
  netstat -ano | findstr :8000
  ```
- **解决方案**: 
  1. 停止占用进程
  2. 或使用其他端口：`uvicorn main:app --port 8080`

### 5.2 防火墙设置

如需从外部访问：

```bash
# Ubuntu/Debian (ufw)
sudo ufw allow 8000/tcp
sudo ufw reload

# CentOS/RHEL (firewalld)
sudo firewall-cmd --permanent --add-port=8000/tcp
sudo firewall-cmd --reload
```

### 5.3 日志查看方法

#### 应用日志

```bash
# 实时查看日志
tail -f app.log

# 或在启动时指定日志文件
uvicorn main:app --log-level info > app.log 2>&1
```

#### Celery 日志 (如使用)

```bash
# 查看 Celery worker 日志
celery -A worker.celery_app worker --loglevel=info
```

### 5.4 数据目录

- **作业存储目录**: `jobs/<job_id>/`
  - `input/`: 上传的输入文件
  - `intermediate/`: 中间处理结果
  - `output/`: 最终输出结果
  - `logs/`: 作业日志
- **注意**: 确保运行用户有写入权限

### 5.5 常见问题排查

| 问题 | 可能原因 | 解决方案 |
|-----|---------|---------|
| 启动失败，端口被占用 | 8000 端口被其他程序占用 | 更换端口或停止占用进程 |
| 导入模块失败 | 依赖未安装 | `pip install -r requirements.txt` |
| Neo4j 连接失败 | Neo4j 未启动或配置错误 | 检查 Neo4j 服务和 URI 配置 |
| LLM 调用失败 | API Key 错误或配额不足 | 检查 OPENAI_API_KEY 设置 |
| 任务一直处理中 | Celery 未启动（生产模式） | 设置 `GEOFUSION_CELERY_EAGER=1` |
| 内存占用过高 | 大文件处理 | 增加系统内存或限制并发数 |

### 5.6 安全建议

1. **生产环境禁止使用 `--reload` 参数**
2. **使用反向代理**（如 Nginx）提供 HTTPS 支持
3. **配置请求超时**，防止资源耗尽
4. **定期清理** `jobs/` 目录下的历史数据
5. **日志轮转**，避免日志文件过大

## 六、项目配置发现的问题与注意事项

### 6.1 配置文件缺失

项目根目录**不存在 `.env.example` 或 `.env.template` 模板文件**，用户需自行创建 `.env` 文件。建议部署时手动创建。

### 6.2 Neo4j 连接无回退机制

代码 `kg/neo4j_repository.py` 中，如果 Neo4j 环境变量配置不完整，会直接抛出异常。虽然 `kg/factory.py` 支持内存模式回退，但 Neo4j 连接失败时**不会自动切换**，需确保配置正确或设置 `GEOFUSION_KG_BACKEND=memory`。

### 6.3 生产部署注意

- 当前配置使用 Celery Eager 模式（默认），适合开发调试
- 生产环境需配置 Redis 并设置 `GEOFUSION_CELERY_EAGER=0`
- 建议使用进程管理器（如 Supervisor）管理 uvicorn 进程

### 6.4 API 端点版本

- v1 API 提供基础融合功能
- v2 API 提供智能体驱动的融合工作流
- v2 功能需要 LLM 配置，否则部分功能不可用
